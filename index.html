JavaWeb

发表于 2017-01-14
就最原始的java web项目而言，最后端是数据库（oracle,sql server,mysql,ect）,数据库语言（SQL）本质上与编程语言不同，

为了实现自动化，就需要一套api连接数据库并能够自动执行sql语句的操作，对于java,对应的是jdbc,而.net，对应的是odbc,但无一列外的，
这种api都比较繁琐，所以对应生出了更新的框架，而其中可以优化的地方是预备sql语句。

连接包括断开数据库之类的操作（对应操作数据库客户端）对应的类和方法都可以封装在一个多个类中（根据需要）放在一个package中，方便使用，并且解耦，因为仅仅是使用。
而数据库中对应的表结构可以对应到所谓entity类中，数据库中的一张表对应一个实体类，并且实现get,set方法，一般都定义为final类型，防止继承。
对应在高级语言中这些类就可以当作一种数据结构用来保存和数据库表中一样的值，方便在高级语言中使用api进行CRUD操作。

对应在dao层，可以在相应的每张表对应的entity类来对应创建dao类,其中实现对应的方法来完成整个数据库的操作，
<但是开始和断开数据库连接以及帐号密码这种不属于现有对应逻辑的部分（或者说与具体的实现CRUD的操作无关，只与数据库操作有关的部分并且重复使用的，封装到db层的方法中，如果发生修改会很方便修改）>,
至此后端数据库逻辑梳理完，

关于前台，html页面需要取值，通过http协议以及一些列更加底层的tcp/ip协议进行传输，同样需要一套api对传输的过程以及值得传递进行控制
servlert就是用来动态的获取值部署到server端动态的生成html页面，并控制一些响应，
由于同样api非常繁琐，对应又出现了jsp,之后又出现了更新的框架。

在之后html页面上的显示就需要用到前端的知识了

oracle

发表于 2017-01-08

安装oracle 11g遇到的问题：

1. 安装orcale11g 出现[INS-07009] 无法加载 BeanStore.

将用于配置jdk的环境变量classpath改为class_path,或者直接删除。

2. 安装完成后使用b/s端登陆，使用普通的scott账号，显示应用程序要求的数据库权限超出了您当前具有的权限，

使用oracle的sqlplus工具使用管理员帐号登录

请输入用户名：as sysdba
请输入用户名：sys
输入口令：（输入你安装时设定的口令）
grant SELECT_CATALOG_ROLE to user（scott），即可正常登陆。

PS:修改密码命令：

alter user scott account unlock;
alter user scott identified by abc123;

查看账户

1、查看当前用户拥有的角色权限信息：select * from role_sys_privs;
2、查看当前用户的详细信息：select * from user_users;
3、查看当前用户的角色信息：select * from user_role_privs;

搭建博客

发表于 2017-01-02

元旦假期折腾了好一会，终于搭起来了这个博客。
搭建博客时参考：
http://lowrank.science/Hexo-Github/
截至目前应该都是能够正确使用的，细节上面已经说的很详细了。

关于搭建过程中遇到的问题：

1.名字一定要是your_name.github.io,这里的your_name指的是自己github账户的名字
2.执行$ ssh-keygen -t rsa -b 4096 -C “your_email@example.com”的这条命令报了git ssh Too many arguments.原因是没有加上双引号。
3.执行一些npm相关的命令时，可能会很长时间下载不下来，所以需要替换成taobao的npm镜像。
4.修改_config.yml这个文件时一定要注意“：”后面有一个空格，否则会导致格式不正确而出现错误。
5.在本地localhost测试的时候默认使用的是4000端口，可能被占用，需要使用命令$ hexo server -p 4001 替换一个可用端口

PS:一些其他的

1.netstat -a 查看本机端口使用情况
2.hexo s –debug 查看网站运行情况
