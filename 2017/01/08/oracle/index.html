发表于 2017-01-08

安装oracle 11g遇到的问题：

1. 安装orcale11g 出现[INS-07009] 无法加载 BeanStore.

将用于配置jdk的环境变量classpath改为class_path,或者直接删除。

2. 安装完成后使用b/s端登陆，使用普通的scott账号，显示应用程序要求的数据库权限超出了您当前具有的权限，

使用oracle的sqlplus工具使用管理员帐号登录

请输入用户名：as sysdba
请输入用户名：sys
输入口令：（输入你安装时设定的口令）
grant SELECT_CATALOG_ROLE to user（scott），即可正常登陆。

3. java.sql.SQLSyntaxErrorException: ORA-00942: 表或视图不存在异常，或者can not get sequence nextval

将表建在了dba权限的用户下，所以jdbc中以normal角色访问数据库就会出问题。

PS:修改密码命令：

alter user scott account unlock;
alter user scott identified by abc123;

查看账户

1、查看当前用户拥有的角色权限信息：select * from role_sys_privs;
2、查看当前用户的详细信息：select * from user_users;
3、查看当前用户的角色信息：select * from user_role_privs;

oracle用户权限

1.system用户只能用normal身份登陆em（b/s）。除非你对它授予了sysdba的系统权限或者syspoer系统权限。
2.sys用户具有“SYSDBA”或者“SYSOPER”权限，登陆em也只能用这两个身份，不能用normal。

详见 http://blog.csdn.net/haiross/article/details/25777335

dual表

dual可以看成一个万能的虚拟表。 eg:select 2+3 from dual;就像得到了一个需要的返回值(可能用于insert语句)。
